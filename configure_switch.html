Configuring Initial Switch Settings

In this lab, we will try to configure a cisco switch using a packet tracer simulator.

Topology

Objective


The Objective of this lab is to Learn how to:


  • Verify the Default Switch Configuration

  • Configure a Basic Switch Configuration

  • Configure a MOTD Banner

  • Save Configuration Files to NVRAM


In this activity, I performed basic switch configurations.
I secured access to the command-line interface (CLI) and console ports using encrypted and plain text passwords.
I alsoconfigured messages for users logging into the switch.
These banners are also used to warn unauthorize users that access is prohibited.

Part 1


Verifying the Default Switch Configuration


  • Step 1: We need to Enter the EXEC Privileged mode of the switches.


    We can access most of the commands in EXEC Privileged mode. However, since numerous privileged commands adjust operating parameters, it's essential that privileged access is password- protected to thwart unauthorized usage. The privileged EXEC command set encompasses commands found in user EXEC mode, alongside the configure command, which grants access to the remaining command modes.


    1. Click S1 and then the CLI tab. Press Enter.


    2. Enter privileged EXEC mode by entering the enable command:

      Switch> enable
      Switch#



      We can see that the prompt changed in the configuration to reflect privileged EXEC mode.

Part 2


Create a Basic Switch Configuration


  • Step 1: Let’s Assign a name to the switch.


    When configuring parameters on a switch, you might need to navigate through different configuration modes. Pay attention to how the prompt alters as we move across the switch.


    Switch# configure terminal
    Switch(config)# hostname S1
    S1(config)# exit
    S1#



  • Step 2: We need to Secure access to the console line.


    To secure access to the console line, let’s access config-line mode and set the console password to “secure”.

    S1# configure terminal
    Enter configuration commands, one per line. End with CNTL/Z.
    S1(config)# line console 0
    S1(config-line)# password secure
    S1(config-line)# login
    S1(config-line)# exit
    S1(config)# exit



    The login command is necessary because for the password checking process to function properly, both the login and password commands are required.



  • Step 3: Verify that console access is secured.


    Exit privileged mode to verify that the console port password is in effect.


    S1# exit
    Switch con0 is now available
    Press RETURN to get started.
    User Access Verification
    Password:
    S1>



  • Step 4: Secure privileged mode access.


    Set the enable password to “cisco”. This password protects access to privileged mode.


    S1> enable
    S1# configure terminal
    S1(config)# enable password cisco
    S1(config)# exit
    S1#



  • Step 5: Verify that privileged mode access is secure.


    Enter the exit command again to log out of the switch.
    Press and you will now be asked for a password:
    User Access Verification
    Password:




    The first password is the console password we configured for line con 0 which is “secure”. Enter this password to return to user EXEC mode.

    Enter the command to access privileged mode. Enter the password which is “cisco”.


  • Step 6: Let’s Configure the encrypted password to secure access to privileged mode.


    The enable password should be replaced with the newer encrypted secret password using the enable secret command. Set the enable secret password to “secret”.


    S1# config t
    S1(config)# enable secret secret
    S1(config)# exit
    S1#

    Note: The enable secret password takes precedence over the enable password. If both are configured on the switch, you need to enter the enable secret password to access privileged EXEC mode




  • Step 7: Now let’s Verify that the enable secret password is added to the configuration file.


    Enter the show running-config command again to verify the new enable secret password is configured.



    Why do we notice a difference in the display of the enable secret password compared to what was configured? The "enable secret" is exhibited in encrypted form for security reasons, while the enable password remains in plain text


  • Step 8: Time to Encrypt the enable and console passwords.


    As we noticed in Step 7, the enable secret password was encrypted, but the enable and console passwords were still in plain text. We will now encrypt these plain text passwords using the service “password-encryption command” .

    S1# config t
    S1(config)# service password-encryption
    S1(config)# exit




    What do you think If we configure any more passwords on the switch, will they be displayed in the configuration file as plain text or in encrypted form? The service password-encryption command encrypts all current and future passwords.

Part 3


Configuring an MOTD(Message of the Day) Banner


The Cisco IOS command set incorporates a functionality enabling you to set up messages visible to anyone logging into the switch. These messages are referred to as "message of the day" (MOTD) banners. Remember to enclose the banner text in quotations or use a delimiter that differs from any character present in the MOTD string

S1# config t
S1(config)# banner motd "This is a secure system. Authorized Access Only!"
S1(config)# exit
S1#



We will encounter the MOTD Banner when attempting to log in to the switch again.

Every switch should have a MOTD banner for several reasons. Primarily, it serves as a warning to unauthorized users that access is prohibited, enhancing security. Additionally, it can be utilized to communicate important messages to network personnel or technicians, such as impending system shutdowns or instructions on who to contact for access-related inquiries



Part 4


  • Step 1: Let us first Verify that the configuration is accurate using the show run command.


  • Step 2: Now we save the configuration file.


    We have completed the basic configuration of the switch. Now back up the running configuration file to NVRAM to ensure that the changes made are not lost if the system is rebooted or loses power.

    S1# copy running-config startup-config
    Destination filename [startup-config]?[Enter]
    Building configuration...
    [OK]



  • Step 3: Examine the startup configuration file.


    Enter the following command and compare it to the running -config. It should be the same


    S1# show startup-config


Part 5


We have completed the configuration on S1. We will now configure S2. Let’s Configure S2 with the following parameters:


  1. Name device: S2
  2. Protect access to the console using the “secure” password.
  3. Configure an enable password of “cisco” and an enable secret password of
  4. Configure a message to those logging into the switch with the following message:
    "Authorized access only. Unauthorized access is prohibited and violators will be prosecuted to the full extent of the law."
  5. Encrypt all plain text passwords.
  6. Ensure that the configuration is correct.
  7. Save the configuration file to avoid loss if the switch is powered down.

Switch> enableb
Switch# config t
Switch(config)# hostname S2
S2(config)# line console 0
S2(config-line)# password secure
S2(config-line)# login
S2(config-line)# enable password cisco
S2(config)# enable secret secret
S2(config)# banner motd "This is a secure system. Authorized Access Only!"
S2(config)# service password-encryption
S2(config)# do copy running-config startup-config




This represents the initial configuration procedure for a switch. Additionally, there are other steps involved, such as enabling SSH and other functions based on the organization's needs.
